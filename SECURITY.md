# Security Policy

## Supported Versions

We provide security updates for the latest stable release of Desktop Calendar Tracking. Older versions may not receive fixes, so please upgrade to the latest version when possible.

| Version | Supported          |
| ------- | ------------------ |
| 1.2.x   | :white_check_mark: |
| < 1.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly. **Do not** open a public issue.

You can report a vulnerability by:

- **Emailing the maintainer** at [richard@richardsblogs.com](mailto:richard@richardsblogs.com). Please include:
  - A description of the vulnerability.
  - Steps to reproduce (if applicable).
  - Any potential impact.
  - Your contact information for follow-up questions.

- **Using GitHub's private vulnerability reporting** (if enabled) by navigating to the repository's **Security** tab and clicking "Report a vulnerability".

We aim to acknowledge receipt of your report within 48 hours and will provide an estimated timeline for a fix. We will keep you informed of progress.

If the vulnerability is accepted, we will work on a patch and release a security update. If declined, we will explain why.

Thank you for helping keep Desktop Calendar Tracking and its users safe!
